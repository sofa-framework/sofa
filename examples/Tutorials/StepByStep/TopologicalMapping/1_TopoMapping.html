Topological Mapping (2/7)


Description

In this step we load a tetrahedral mesh using a mesh file and dynamique topological components.For details concerning differences between static and dynamic topologies, refere to topological tutorials.

Key points
  • A meshLoader to read mesh file. (not necessary in regular mesh)
  • A mechanicalObject with geometry point positions.
  • Dynamic topological components to store the volume mesh.


Option available in the TetrahedronSetGeometryAlgorithms component is used to display topological element (tetrahedra here).
Results

In this scene we only have the tetrahedral mesh in terms of topology and geometry. No interactions are possible at this step.
