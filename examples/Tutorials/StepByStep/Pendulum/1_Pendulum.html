Creation of a Pendulum (2/8)

Description

In the previous scene, no solver was present, so no simulation was done! We add an ODE solver in the Root node of the Simulation. We provide several ODE solvers, with different integration scheme: take a look to our tutorials on Solvers, and see also all the components available in the section OdeSolver of the Modeler.
Key points

We will use the classic explicit euler solver EulerSolver to update particle positions and velocities at each time step: In Sofa, a solver acts on all the objects placed below in the hierarchy.

Results

This time, when you animate the scene, the particule falls along the direction of the gravity.
The gravity is specified in the root node of the simulation. We use a default value of (0, 0, -9.81). Changing the gravity on child nodes of the simulation will not do anything as at each time step, the gravity, and other parameters (time, dt...) is overwritten using the value specified in the root node.
The component Gravity lets you specify a particular gravity for a node and all its children. There is an example available in the Modeler when you select the Gravity component in the list of components.
