Introduction

This is a starting point for writing a plugin. To customize it, you can:
  1. Copy the EmptyPlugin directory and rename it, for example, MyPlugin
  2. Open a command line tool in the MyPlugin directory and use the script utils/changeClassName.sh  to replace EmptyPlugin with MyPlugin in all the file names and their content:
    utils/changeClassName.sh EmptyPlugin MyPlugin
    cd examples
    ../utils/changeClassName.sh EmptyPlugin MyPlugin
    cd ../doc/
    ../utils/changeClassName.sh EmptyPlugin MyPlugin
    doxygen Doxyfile

  3. To insert it in the SOFA development project, you can:
    1. add the following line to your sofa-local.prf
      DEFINES += SOFA_HAVE_EmptyPlugin
    2. add the following line to Sofa.pro
      contains(DEFINES, SOFA_HAVE_EmptyPlugin) { usePlugin(EmptyPlugin) }
    3. if your plugin depends on other plugins, edit file EmptyPlugin-dependencies.prf . See sofa-dependencies.prf for examples of how dependencies are defined.

Project structure

  • the C++ code is in the root directory
  • the documentation starts in doc/index.html and it is linked to the code documentation. The main page of this documentation is defined in file initEmptyPlugin.h.
  • the examples are in examples/ . The first one is EmptyPlugin.scn, and when runSOFA opens it, an html page pops up, with a link to the plugin documentation.
